C.M.S.

Content Management System

= Système de gestion de contenu

C.M.S.

  • Solution prête à l'emploi (paramétrage)
  • Connaissances techniques limitées (en théorie)
  • Séparation contenu (données)/design (thèmes interchangeables)
  • Séparation front-office/back-office
  • Conception modulaire (plugins, extensions)

Avant les C.M.S.

  • Sites statiques (HTML sans BDD)
  • Difficultés de mises à jour
  • Alternatives : développement sur-mesure, frameworks

Fonctionnalités

  • Interface de saisie de contenu (Wysiwyg)
  • Gestion des menus
  • Gestion des médias
  • Gestion des utilisateurs (rôles, droits)
  • Gestion des menus
  • Utilitaires divers (pagination, référencement, cache, connecteurs etc.)

Workflow (exemple)

Modules et thèmes

  • Natifs, gratuits, payants, sur-mesure
  • Installation/désinstallation simple
  • Modifications graphiques en live (ajout d'un bloc)
  • Thèmes "responsive"

Qu'est-ce qu'un contenu

  • Un contenu : texte (article), médias (images, vidéo), fichier (PDF...)
  • Types de contenu prédéfinis : article, page, média, produit (e-commerce)
  • Types de contenu créé par l'utilisateur (par exemple : annonce immo)
  • Catégorisation des contenus (taxonomies, mots-clés)
  • Métadonnées (auteur, modifications etc.)

Usages

  • Blogs
  • Sites vitrines/corporate (T.P.E., P.M.E.), portails
  • Portails, Intranet, extranet
  • E-commerce
  • Forums, réseaux sociaux, etc.

Technologies

  • PHP (CMS "purs") : Wordpress, Joomla, Drupal, Typo3, EzPublish
  • PHP : E-commerce : Prestashop, Magento...
  • Autres langages : Java, Microsoft Sharepoint, NodeJS etc.

Critères de choix

  • Selon l'expression de besoins (fonctionnalités)
  • Compétences équipes
  • Communauté
  • Coûts (comme d'habitude !)

Avantages

  • Prêt à l'emploi (en théorie seulement)
  • Simplicité de gestion des contenus
  • Couverture fonctionnelle (des modules pour tout ou presque)
  • Bonnes pratiques implémentées
  • Gain de temps
  • Coût modéré (selon techno)

Inconvénients

  • Courbe d'apprentissage : compétences spécifiques (fonctionnelles, techniques)
  • Complexité fonctionnelle et technique, respect de l'architecture
  • Intégration d'autres technos parfois difficile
  • Fonctionnalités inutiles, trop d'extensions à installer
  • Fréquence des mises à jour (noyau, modules, thèmes, traductions)
  • Sécurité : code connu, failles publiées
  • Choix difficiles (solution, technos, modules, thèmes etc.)

Wordpress

  • 60% des CMS, 35% des sites toutes technos confondues

E-commerce : problématiques

  • Gestion des produits (déclinaisons, informations, photos etc.)
  • Prix, TVA/taxes, facturation
  • Livraison : frais, logistique, SAV
  • Réglementation (produits autorisés, comptabilité/archivages, données personnelles)
  • Moyens de paiement, paiement en ligne, sécurité
  • Intégration au S.I. de l'entreprise (CRM/ERP, production, logistique, compta)
  • E-marketing
